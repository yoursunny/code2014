创建列表 - PageGenerator帮助

决定列表内容

考虑列表应该包含插入那些主题的页面，然后记住并执行这个想法
PageGenerator不会自动将页面插入列表

建议：
仔细检查创建的页面，防止任何错误、排版混乱、错别字，细节会决定访问者对站点的印象
仅将符合列表主题且经过检查确保没有错误的页面加入列表

决定列表输出格式

HTML
输出为单独的网页，通常直接作为首页/栏目页或者在首页/栏目页用<iframe>标签引用
JavaScript
输出为.JS脚本文件，在客户端执行；可以在插入<script>直接输出列表内容，也可以在网页中包含<div>标签、<script>插入在网页最后(以便提高网页打开速度)、修改<div>的内容
注意：JavaScript列表通常不能被搜索引擎解释，请确认页面同时被加入了其他列表
XML
输出为Atom聚合源，可以用Atom阅读器(如Foxmail、BlogLines、Google Reader)阅读和聚合

制作HTML页面

当需要输出成HTML或JavaScript格式(通常不能够同时输出这两种格式)时，请用Dreamweaver等软件制作好列表的HTML页面，标题、摘要、链接等可以随便输入；请确保标题字数过多等情况下页面能正常显示(可以使用CSS的相关属性)。对于HTML，制作一个小小的网页，准备在大的网页中用<iframe>标签嵌入；对于JavaScript，在大的网页中添加一个<div>或<td>等标签，设置好一个唯一的id="..."属性，并在标签内制作列表的格式。考虑好列表应该显示多少条目；HTML列表在一页显示不下时会自动创建更多的页，JavaScript列表则只能显示第一页的内容。

提取重复部分

如果准备输出HTML文件，应包含<html><head><body>等；如果准备输出JavaScript文件，则应去除这些标签，只保留<div id="..."></div>之间的内容(不含<div>与</div>)。
阅读列表HTML代码，将其中每个条目都有的代码提取出来，将原来位置的代码删除并改成%l%。
将提取出来的代码只保留一个条目的，然后替换以下代码(如果不需要某项，不插入相关代码即可)：

内容 替换为 输出样例 备注
链接地址 <a href="%link%" ...> demo 绝对URL
文章标题 %title%

本站正式启动

 只有文字
不含样例中<h1>
日期-年 %date_y% 2007年02月28日 四位数字
日期-月 %date_m% 有前导0
日期-日 %date_d% 有前导0
时间-时 %time_h% 23:59:59 24小时制
有前导0
时间-分 %time_m% 有前导0
时间-秒 %time_s% 有前导0
作者 %author% 管理员 作者姓名
不是用户名
摘要 %summary% 欢迎来到本站  

将重复部分保存为list.列表名称.inc文件，注意“编码”应该选择UTF-8。

完成列表页面

把已经将重复部分改成%l%的剩余代码做以下替换修改(如果不需要某项，不插入相关代码即可)：

内容 替换为 输出样例 备注
列表名称 %ltitle% 我的列表  
日期-年 %ldate_y% 2007年02月28日 四位数字
日期-月 %ldate_m% 有前导0
日期-日 %ldate_d% 有前导0
时间-时 %ltime_h% 23:59:59 24小时制
有前导0
时间-分 %ltime_m% 有前导0
时间-秒 %ltime_s% 有前导0
总条目数 %lcount% 22  
每页最多条目数 %lpageentry% 4  
总页数 %lpages% 6  
站点根目录路径 <a href="_root_/images/a.gif" ... <a href="../../images/a.gif" ... 第二页起与第一页不在同一目录，因此请勿自行使用相对路径
上一页链接 <a href="%prev%" style="%prevd%" ...> << 没有上一页/下一页时，%prevd%/%nextd%会被设为display:none;从而隐藏链接
下一页链接 <a href="%next%" style="%nextd%" ...> >>
本页条目数 %pageentry% 2  
本页页码 %pn% 6  

将替换完成的代码保存为list.列表名称.htm文件，注意“编码”应该选择UTF-8。

书写信息文件

html=/mylist.htm  输出HTML文件路径，不输出就不写本行
atom=/mylist.xml  输出Atom文件路径，不输出就不写本行
js=/mylist.js  输出JavaScript文件路径，不输出就不写本行
jsdiv=div_mylist  JavaScript需设置的<div>的id属性值，注意大小写
                  输出JS但无div设置的，就会在调用位置直接document.write
max=4  每页输出条目数
title=我的列表  列表标题
user=sunny  有权编辑本列表的用户名(站点管理员不会自动获得列表管理权)
user=love   多个管理员可写多行

将写好的信息文件保存为list.列表名称.txt文件，注意“编码”应该选择UTF-8。

创建Atom数据文件

列表数据本身也保存在Atom文件中，只需下载空白文件，保存为list.列表名称.xml即可。

上传列表配置

将创建的四个文件上传到站点的/_PageGenerator_/目录下，一个列表就创建好了。
