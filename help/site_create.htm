创建新站点 - PageGenerator帮助

步骤

  1. 定义站点发布途经
  2. 安装基本文件

1 定义站点发布途径

站点由/conf/site.*.axd文件定义
仿照以下例子写好site.*.axd文件，保存在PageGenerator管理程序的/conf/目录下(*是管理程序中显示的站点名称)

以下例子中，PageGenerator管理程序安装在http://www.mydomain.com/p/(202.120.54.34服务器E:\www1目录)
红色是不可以修改的部分，原样填写；黄色是注释，不能输入

1.1 本地发布(同一域名下的虚拟目录)

例：在http://www.mydomain.com/mydir/1/创建网站

site.*.axd书写如下：

LOCAL
/mydir/1  虚拟目录路径，前面有/，最后没有/

例：在http://www.mydomain.com/创建网站

site.*.axd书写如下：

LOCAL
/  虚拟目录路径，前面有/，最后没有/

1.2 本地发布(同一服务器、不同域名下)

例：在http://pub.mydomain.com/(服务器F:\www2\目录)创建网站
(www.mydomain.com站点的ASP.NET启动账户需要对F:\www2\目录有读取、写入、删除等NTFS权限)

site.*.axd书写如下：

SERVER
F:\www2  绝对路径，最后没有\

1.3 远程发布(不同服务器、通过FTP)

例：在http://www2.mydomain.com/love/(202.120.61.131服务器)创建网站

在202.120.61.131安装FTP服务(Microsoft FTP Service、Serv-U等软件)
设FTP服务安装在53721端口，用户名love，密码inspiron
FTP权限为读取文件、写入文件、列出目录、创建目录、删除目录
设FTP登录后的/public_html/love/目录对应http://www2.mydomain.com/love/

site.*.axd书写如下：

FTP
202.120.61.131  服务器IP地址或域名
53721  端口，如果使用默认端口请写21
/public_html/love  FTP目录，最后没有/
love  用户名
inspiron  密码

FTP发布方式会使PageGenerator服务器与网站服务器产生大量的流量
站点的一切内容均是存储在FTP服务器上，会进行反复的上传、下载
如果两台服务器间网络连接比较慢，请不要使用FTP发布方式

不使用FTP发布方式，需要发布到远程服务器，无法在该服务器安装PageGenerator时
可以考虑在本地服务器创建一个“镜像”站点，在该站点上进行管理
然后通过增量打包功能下载修改过的文件，上传到远程服务器解压缩
(解压功能可选用ASP.NET或PHP版本的解压工具)

2 安装基本文件

基本文件应该上传到发布服务器(站点所在服务器)的站点目录中
基本文件给出了站点的更多信息，也使一些功能能够快速实现

  1. 书写_PageGenerator_.txt站点信息文件，放在站点根目录
    user=sunny  站点总管理员用户名，这些管理员可管理全站
user=admin  多个总管理员可写多行
user=root
name=我的站点  站点名称
url=http://www.yourdomain.com  站点访问地址，http://开头，最后没有/
  2. 解压缩静态程序包在站点/_PageGenerator_/目录(请保持目录结构)
  3. 选择合适的动态功能包，解压缩在站点/_PageGenerator_/目录(请保持目录结构)
    请根据发布服务器支持的脚本技术选择：
    ASP.NET+MSSQL 高速高效，需要SQL2005数据源服务
    ASP.NET+TXT+ACCESS 大量访问时速度慢
    ASP+ACCESS 速度慢，仅能支持少量访问
    PHP+MySQL 高速高效，需要MySQL数据源服务
